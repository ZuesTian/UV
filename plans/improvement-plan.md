# 光谱面积计算工具 — 全面评估与改进方案

## 一、现有代码评估

### 1. 架构问题

- 单文件 904 行，数据模型/IO/业务逻辑/UI 全部混在 [`SpectrumApp`](spectrum_gui.py:56) 一个类里
- `self.x`/`self.y` 与 `self.documents` 双重持有数据，切换文件时需手动同步多个 StringVar
- 计算逻辑 [`compute_area`](spectrum_gui.py:805) 依赖实例变量，无法独立测试

### 2. 多文件模式痛点

- [`load_files`](spectrum_gui.py:325) 每次覆盖已有文件列表，无法追加
- 无批量积分、无汇总表格、无导出功能
- 多谱对比只能叠加曲线，无数值对比

### 3. GUI 布局问题

- 右侧边栏 360px 塞了多谱对比+饼图+峰记录+当前结果，过于拥挤
- 顶部工具栏文件操作和视图范围混在一起
- 底部积分范围+计算+记录峰挤在一行

## 二、改进方案

### A. GUI 布局重构 — 左右双面板

```
+------------------------------------------------------------------+
| 工具栏: [打开文件] [追加文件] [文件下拉框] | 已加载: N 个文件      |
+-------------+----------------------------+-----------------------+
| 左侧面板     |                            | 右侧面板              |
|             |                            |                       |
| [文件管理]   |       图表区域              | [峰记录]              |
|  文件列表    |    Matplotlib Canvas       |  记录列表              |
|  勾选对比    |                            |  基准峰选择            |
|  全选/清空   |                            |  对比结果文本          |
|             |                            |  饼图                 |
| [积分计算]   |                            |  [删除选中]            |
|  范围输入    |                            |                       |
|  [计算当前]  |                            | [汇总与导出]           |
|  [批量计算]  |                            |  [查看汇总]            |
|  当前结果    |                            |  [导出 CSV]            |
|  [记录峰]   |                            |                       |
+-------------+----------------------------+-----------------------+
| 状态栏: 视图范围 X最小/X最大/Y最小/Y最大 [应用视图] [重置视图]     |
+------------------------------------------------------------------+
```

**关键设计决策：**

1. **左侧面板**（约 280px）：文件管理 + 积分计算，操作流程从上到下自然衔接
2. **右侧面板**（约 300px）：峰记录 + 对比 + 汇总导出，结果展示区
3. **中间图表区域**：自适应填充剩余空间，比现在更大
4. **顶部**：精简为纯文件操作栏
5. **底部**：视图范围控制移到状态栏位置

### B. 功能流程优化

#### B.1 文件追加模式
- [`load_files`](spectrum_gui.py:309) 新增 `append` 参数，默认追加
- 路径去重避免重复加载
- 保留"清空并重新加载"选项

#### B.2 批量积分
- 左侧面板新增"批量计算所有文件"按钮
- 对所有已加载文件使用当前积分范围执行 `compute_area`
- 结果自动记录到各文件的 peak_records

#### B.3 汇总表格与导出
- 右侧面板新增"查看汇总"按钮，弹出 Toplevel 窗口
- Treeview 表格：文件名 | 峰名 | 范围 | 扣基线面积 | 仅正面积
- "导出 CSV"按钮将汇总数据写入文件

#### B.4 数据模型改进
- 新增 `PeakRecord` dataclass 替换 `peak_records: list[dict]`
